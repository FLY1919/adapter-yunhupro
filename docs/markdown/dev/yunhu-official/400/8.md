# 视频上传

## 请求URL

`https://chat-go.jwzhd.com/open-apis/v1/video/upload?token=016ec8e4d17a4fbda3ccb0e99b44aca0`

### 请求类型

`POST`

### 请求头

```bash
Content-Type: "multipart/form-data"
```

### 请求参数

| 字段  | 类型 | 是否必填 | 说明                                            |
| ----- | ---- | -------- | ----------------------------------------------- |
| video | file | 是       | 视频内容，仅限单个视频，上传视频大小不超过 20MB |

### 响应内容

| 字段 | 类型   | 说明                   |
| ---- | ------ | ---------------------- |
| code | int    | 响应代码               |
| msg  | string | 响应信息，包括异常信息 |
| data | Object | 返回数据               |

## 完整示例

```bash
curl --location 'https://chat-go.jwzhd.com/open-apis/v1/video/upload?token=016ec8e4d17a4fbda3ccb0e99b44aca0' \
--header 'Content-Type: multipart/form-data' \
--form 'video=@"D:/video/aaa/bbb.mp4"'
```

#### 视频上传响应示例

```json
{
  "code": 1,
  "data": {
    "videoKey": "016ec8e4d17a4fbda3ccb0e99b44aca0xx"
  },
  "msg": "success"
}
```

## 如何获取访问地址

上传成功后返回的 `videoKey` 不是最终访问地址，不能直接拿来拼接视频 URL。`videoKey` 只是上传成功后的返回值，真正的访问地址需要根据视频内容重新计算哈希。

视频访问地址的生成规则如下：

`https://chat-video1.jwznb.com/{videoHash}.mp4`

其中 `videoHash` 是对视频文件内容计算得到的哈希值，不是接口返回的 `videoKey`。

例如，上文我们上传的是 `D:/video/aaa/bbb.mp4`，那么在代码里会先读取并处理这个视频文件，再对最终视频内容做哈希计算，得到对应的 `videoHash`，然后拼接成：

`https://chat-video1.jwznb.com/{videoHash}.mp4`

如果你要找当前实现对应的源码，可以参考：

- [VideoUploader.ts](https://github.com/FLY1919/adapter-yunhupro/blob/main/src/uploader/VideoUploader.ts)
- [AudioUploader.ts](https://github.com/FLY1919/adapter-yunhupro/blob/main/src/uploader/AudioUploader.ts)

其中音频消息不是直接上传音频文件，而是会先转换为视频，再按同样的哈希规则生成最终访问地址。
